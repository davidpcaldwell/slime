  • run
    Launches a subprocess using the operating system.
    Arguments
    1. (array of string) The set of command tokens to pass to the underlying operating system.
    2. has properties:
      • stdin
        A native java.io.InputStream, which will be attached to the subprocess as its standard input stream.
      • stdout
        A native java.io.OutputStream, which will be attached to the subprocess as its standard output stream.
      • stderr
        A native java.io.OutputStream, which will be attached to the subprocess as its standard error stream.
      • environment
        A set of names and values that will be passed to the subprocess as its environment.
      • work
        Specifies the working directory to use for the subprocess; can be either a string specifying the full path, or a native java.io.File object representing the directory.
      • onExit
        A function that will be invoked when the subprocess terminates, and will be provided with information about the result.
        If this property is omitted, a default will be used that throws an exception if the exit status of the subprocess is not zero or the subprocess cannot be launched.

          • environment

          • workingDirectory

  • environment
    object An object describing the operating system process environment under which this script was executed. Each variable defined in the environment is represented as a property in this object, with its name being the name of the property and its value being the value of that property. This object uses JavaScript semantics, and thus this object uses case-sensitive names, even if the underlying operating system does not.
  • properties
    (experimental) An object that attempts to allow the use of JavaScript semantics to access Java system properties. For example, the java.home system property is referenced as properties.java.home. This value can be examined using String(properties.java.home) to obtain its value as a string.
