The SLIME project provides tools for JavaScript development on several platforms, including the Java platform (using the Rhino, Nashorn, or GraalVM JavaScript engines), Java servlet-based servers, and the browser.

Platforms

jsh: a JavaScript application environment that can access the Java platform.

Java servlets: a JavaScript environment for authoring Java servlets

Browser: a SLIME environment for the browser

JXA: a SLIME environment for use in macOS automation

Tools

Command-line tools

Testing

  • Running individual definitions (JSAPI pages or Fifty definitions)
    • tools/fifty.bash test.jsh file.fifty.ts [--part part]
    • tools/fifty.bash test.browser [--interactive] [--chrome:data pathname] [--chrome:debug:vscode] file.fifty.ts [--part part]: Invokes a script to run Fifty browser tests; see the script for command-line argument definitions.
    • jsh: ./jsh.bash jsh/test/suite.jsh.js -definition definition [-part part]
    • Browser definition page: ./jsh.bash loader/browser/test/suite.jsh.js -definition pathname [...]
  • Running browser suite: ./jsh.bash loader/browser/test/suite.jsh.js -suite pathname [-base directory] [-interactive]
  • loader/api/tools/update.jsh.js [-base directory]: Inlines or updates the definition files for a library that does not itself contain SLIME (and thus cannot link to the SLIME definition files in loader/api).

Documentation

  • tools/documentation.jsh.js: Serves the documentation for project, in a dedicated Chrome browser if Chrome is found. Typedoc documentation will be generated if necessary and served for both the top-level repository and subrepositories at local/doc/typedoc (it will be re-generated for each request if --watch is used).
    • --base directory: top-level directory of the project.
    • --host hostname: hostname to use when serving pages.
    • --index path: relative path to the index page; defaults to README.html.
    • --watch: if present, causes documentation to be re-generated for every HTML page requested; helpful when authoring docunmentation.
    • --chrome:id name: the private Chrome directory to use (under local/chrome); defaults to documentation, or document if --watch is set).

Other

  • rhino/http/servlet/tools/serve.jsh.js [--chrome:data location] [--chrome:debug:port number] [--index path-to-index] directory: Serves a directory on an ephemeral port. If --chrome:data is provided, Chrome will be opened to browse the directory, starting at the path specified by --index (default: index.html). If --chrome-debug-port is specified, the opened Chrome will allow debuggers to connect on the given port.
  • loader/api/tools/update.jsh.js [-base directory]: Inlines or updates the definition files for a library that does not itself contain SLIME (and thus cannot link to the SLIME definition files in loader/api).

wf

SLIME also provides tools for projects to implement commands related to the software lifecycle, including a top-level script that can be invoked (at tools/wf) and an associated API (in the form of a jsh plugin).
